Measure Claude Code's baseline token overhead at component level and map the complete system prompt structure. This produces the foundational measurements that all Phase 2 experiments will reference as baselines.

Purpose: Without knowing the exact token cost of each component (system prompt, tools, CLAUDE.md, memory, MCP), we cannot accurately measure the impact of optimization techniques. This plan creates those baseline numbers.
Output: Baseline overhead measurements (JSON + markdown report), per-tool token costs, system prompt structure map.
</objective>

   The script uses progressive payload building (add one component at a time, measure the delta) to isolate per-component costs. Per D-07, each measurement is repeated N times (default 3) to establish variance.

   **Components to measure (per D-05):**
   - Empty baseline: `count_tokens(messages=[{"role": "user", "content": "Hello"}])` -- this is the absolute minimum
   - System prompt: Add Claude Code's system prompt text (obtain from Piebald-AI repo reference or use a representative placeholder). Measure delta.
   - Built-in tool schemas: Add the 7 built-in tool definitions (Read, Edit, Bash, Grep, Glob, Write, Agent). Measure delta for all tools together AND for each individually.
   - CLAUDE.md overhead at different levels:
     - Small CLAUDE.md (~100 tokens of instructions)
     - Medium CLAUDE.md (~500 tokens)
     - Large CLAUDE.md (~2000 tokens)
     Measure delta for each.
   - Memory (MEMORY.md): Simulate a typical MEMORY.md content (~200 tokens). Measure delta.
   - MCP server overhead: Measure with 0 MCP servers, then with sample MCP tool definitions (1 server with 3 tools, 3 servers with 10 tools total). Measure delta.
   - Environment info: Measure overhead of environment context (OS, shell, cwd, git status).
