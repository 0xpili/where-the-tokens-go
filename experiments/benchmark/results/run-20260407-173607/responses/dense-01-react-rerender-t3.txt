Shared state upstream. When a button in Component A updates state — whether in a parent, context, or store — React re-renders all components that consume that state, including yours.

**Common causes:**

- **Parent state** — parent holds state, both components are children → parent re-renders → both children re-render
- **Context** — your component calls `useContext(SomeContext)` → any update to that context triggers re-render
- **Global store** (Redux, Zustand, Jotai) — your component subscribes to a slice that the button's action updates

**How to diagnose:**

```jsx
// Temporarily add this to your component
console.log('re-rendered', new Date());
// Then use React DevTools Profiler → "Why did this render?"
```

**Fixes, in order of preference:**

1. **Narrow context subscriptions** — split contexts so your component only consumes what it needs
2. **Selector granularity** — in Zustand/Redux, select only the specific field, not the whole store slice
3. **`React.memo`** — wraps component, skips re-render if props haven't changed (won't help if the re-render comes from hooks inside)
4. **`useMemo` / `useCallback`** — stabilize values/functions passed as props

The root fix is almost always: make the shared state more granular so fewer components depend on it.
