I don't have filesystem tools available in this context, so I'll deliver the Dockerfile directly.

```dockerfile
# syntax=docker/dockerfile:1

# ─── Stage 1: build ──────────────────────────────────────────────────────────
FROM node:22-alpine AS builder

WORKDIR /app

# Install deps first for better layer caching
COPY package.json package-lock.json ./
RUN npm ci

# Copy source and build (adjust if not using TypeScript)
COPY . .
RUN npm run build


# ─── Stage 2: production ─────────────────────────────────────────────────────
FROM node:22-alpine AS production

ENV NODE_ENV=production
WORKDIR /app

# Install only prod deps
COPY package.json package-lock.json ./
RUN npm ci --omit=dev && npm cache clean --force

# Copy compiled output from builder
COPY --from=builder /app/dist ./dist

# Drop root — run as built-in unprivileged user
USER node

EXPOSE 3000
CMD ["node", "dist/index.js"]
```

Pair with a `.dockerignore`:

```
node_modules
dist
.git
*.test.*
coverage
.env*
```

**Why each choice reduces size:**

| Decision | Effect |
|---|---|
| `node:22-alpine` base | ~50 MB vs ~350 MB Debian |
| `npm ci --omit=dev` in final stage | strips devDependencies entirely |
| `npm cache clean --force` | removes npm's local cache layer |
| Only copy `dist/` from builder | source files never enter prod image |
| `.dockerignore` | prevents `node_modules` from bloating build context |

Typical result: **~160 MB** vs **~1 GB** for a naive single-stage image.

**Adjust these if needed:**
- No TypeScript? Replace the `npm run build` + `dist/` references with your actual entry point
- Prisma or native addons? Add `RUN apk add --no-cache openssl` (or similar) before `npm ci` in the production stage
- Tighter still? Swap `node:22-alpine` for `gcr.io/distroless/nodejs22-debian12` in the production stage — removes shell entirely (~10–20 MB more savings, but no `docker exec` shell access)
