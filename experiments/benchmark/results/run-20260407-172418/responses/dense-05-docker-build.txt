**Plan: Multi-stage Dockerfile for Node.js API**

```dockerfile
# Stage 1: deps — install ALL deps (including devDeps needed for build)
FROM node:22-alpine AS deps
WORKDIR /app
COPY package*.json ./
RUN npm ci

# Stage 2: build — compile TypeScript / run bundler
FROM node:22-alpine AS build
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN npm run build

# Stage 3: prod-deps — install only production deps (clean slate)
FROM node:22-alpine AS prod-deps
WORKDIR /app
COPY package*.json ./
RUN npm ci --omit=dev

# Stage 4: final — minimal runtime image
FROM node:22-alpine AS final
RUN addgroup -S appgroup && adduser -S appuser -G appgroup
WORKDIR /app

COPY --from=prod-deps /app/node_modules ./node_modules
COPY --from=build /app/dist ./dist
COPY package.json ./

USER appuser
EXPOSE 3000
ENV NODE_ENV=production

CMD ["node", "dist/index.js"]
```

And a companion `.dockerignore`:

```
node_modules
dist
.git
*.md
*.log
.env*
coverage
.nyc_output
Dockerfile*
docker-compose*
.dockerignore
```

---

**Why each stage exists:**

| Stage | Base | Purpose |
|---|---|---|
| `deps` | `node:22-alpine` | `npm ci` with dev deps (needed for TypeScript compiler etc.) |
| `build` | `node:22-alpine` | `tsc` / bundler runs here; output → `dist/` |
| `prod-deps` | `node:22-alpine` | Fresh `npm ci --omit=dev` — no build tools leaked |
| `final` | `node:22-alpine` | Only `dist/` + prod `node_modules`; non-root user |

**Size drivers addressed:**
- `alpine` base → ~5MB vs ~900MB for full Debian
- `prod-deps` stage prevents devDependencies from reaching the final image
- `.dockerignore` prevents source, tests, and git history from entering build context
- Non-root user (`appuser`) for least-privilege runtime

**Variants to consider:**
- Replace `node:22-alpine` final stage with `gcr.io/distroless/nodejs22-debian12` for an even smaller, shell-less image (harder to debug but more secure)
- If no TypeScript/build step, drop `build` stage and copy `src/` directly
