Shared state. If both components subscribe to the same state (Context, Redux store, Zustand slice, etc.), any update triggers re-renders in all subscribers — even unrelated ones.

Common causes:

1. **Context too broad** — putting fast-changing state (e.g. click counter) in a high-level context that many components consume
2. **Parent re-renders** — button's click updates parent state, parent re-renders, child re-renders as collateral
3. **Redux/Zustand selector too wide** — `useSelector(state => state)` instead of `useSelector(state => state.specificSlice)`

**Diagnose:** Install React DevTools, enable "Highlight updates when components render." Click the button and see what lights up.

**Fix depends on cause:**
- Narrow your context (split into smaller contexts, or use `useMemo` on the value)
- Wrap child in `React.memo` to skip re-renders when its props didn't change
- Tighten selectors so component only subscribes to what it actually needs
