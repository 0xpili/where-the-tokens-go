The most common cause of "token expired" on a fresh token is **clock skew** — if the server validating the token has a clock behind the server that issued it, the token's `iat`/`nbf` can appear to be in the future, or the `exp` calculation goes wrong.

Other likely culprits:
- `exp` set in **milliseconds** but validated against **seconds** (or vice versa) — e.g., `Date.now()` vs `Math.floor(Date.now() / 1000)`
- `exp` accidentally set to `iat` or a fixed past timestamp
- Timezone mismatch (local time used instead of UTC)
